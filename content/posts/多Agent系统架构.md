# 我搭了一套多 Agent 系统管理日常开发，聊聊架构

我是全栈开发，手里项目太多，时间永远不够。

每天重复做的事：检查服务有没有挂、刷技术新闻找选题、同一篇文章发到 5 个平台、在不同工具之间管理任务。

cron 和 bash 脚本能解决一部分，但我想做得更彻底。搭了一个多 Agent 系统，三个 Agent 各管一摊。

## 架构

```text
           我（定目标、审结果）
                │
                ▼
        🎯 小Z (Commander)
        主 Agent，拆解任务、调度
           ┌────┴────┐
           ▼         ▼
     🔍 巡查      🔨 营造
     监控、巡检    写代码、搭项目
```

## 小Z：指挥官

日常交互的入口。我说"帮我规划一个新项目"，它会查现有项目库、对照我的目标、出方案、把调研丢给巡查，把实现丢给营造。

它读过我的知识库、目标清单、过往决策。了解我的偏好——Go 优先于 Python，喜欢干净架构，讨厌过度设计。不是通用助手，是针对我定制过的。

## 巡查：安静的守卫

靠心跳机制运行，每隔几分钟自动 check-in。

做什么：服务监控、数据采集、定时任务、异常检测。

设计哲学：沉默就是一切正常。我不会收到"一切 OK"的消息，出了问题巡查才开口。

```text
心跳（每 5 分钟）：
  检查服务  → 正常 → 沉默
  检查队列  → 正常 → 沉默
  检查数据  → 异常 → 立即通知我
```

最糟糕的 Agent 是那种不停给你发状态更新的。

## 营造：代码工匠

需要写代码、搭项目、修 bug 的时候交给它。作为后台子 Agent 运行：接到任务，开一个独立会话，干完了通知我。

一个真实的例子——我让小Z帮我规划一个内容分发工具，小Z拆解完任务后派营造去做：搜 GitHub 上的开源项目，对比了 8 个方案，设计技术架构，写提案文档，git commit。

3 分钟。我花 5 分钟 review，搞定。

## 通信机制

```text
我 → 小Z → 分派任务 → 巡查 / 营造
营造 → 独立会话工作 → 完成后汇报
巡查 → 心跳轮询 → 只在异常时告警 → 我
```

每个 Agent 有独立记忆（`MEMORY.md` + 每日日志）、独立人格（`SOUL.md`）、独立工作空间。但它们共享一个东西：我的个人知识库。

## Soul 文件

这部分比较有意思。每个 Agent 有一个 `SOUL.md` 定义它是谁：

小Z 的定义：逻辑驱动，INTJ 人格，追求效率，终极目标是被动收入和财务自由。

巡查的定义：任务导向，强执行力，异常立即上报，正常就闭嘴。

营造的定义：开发助手，代码洁癖，偏好用 Zoe 自己的框架。

Agent 行为稳定不是靠每次写更好的提示词，而是靠固定的人格文件。调这些描述也花了不少时间。

## 个人知识库

三个 Agent 都能访问我的个人知识仓库，一个结构化的 Markdown repo：

```text
me/
├── ⏰diary/       # 每日记录
├── 💡ideas/       # 产品想法、规划
├── 📖posts/       # 博客草稿
├── 📝notes/       # 技术笔记
├── 🎯goal/        # 人生目标
├── 💻codes/       # 代码片段
├── 📚reads/       # 读书笔记
└── 🔖marks/       # 书签
```

小Z 做决策时查我的目标。巡查读任务配置知道该监控什么。营造搭项目时参照我笔记里的编码规范。知识库是它们共享的大脑。

## 日常用途

每天：通过小Z管理任务、快速问答和决策。

每周：内容规划和起草、通过营造做代码审查和项目开发、进度复盘。

全自动：服务监控、队列检查、数据采集。

## 成本

- LLM API 调用：每月 $30-50 左右，主力模型是 Claude
- 服务器：$0，跑在 MacBook 上
- 搭建时间：大约 2 天
- 每周省下：15 小时以上

## 踩过的坑

**从一个开始。** 我一开始只有小Z。手动检查太烦了才加巡查，重复性开发太多才加营造。别第一天就搞三个。

**人格文件很关键。** 没有 SOUL.md 的话，Agent 行为忽高忽低。人格文件给了一个锚点，让行为可预期。

**沉默即正常。** 巡查的"出了问题才说话"原则省了很多注意力。

**记忆各管各的。** 各自维护记忆意味着各自在自己领域积累经验。巡查记得历史故障，营造记得以前的架构决策。比一个共享的大 blob 好用得多。
