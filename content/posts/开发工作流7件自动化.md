# 我自动化了开发工作流中的 7 件事

每周省 15 小时以上。没用什么花哨的 SaaS，就是 AI agent 加脚本。

## 1. Code Review 预审

AI agent 在我看 PR 之前先过一遍。标出逻辑漏洞、遗漏的边界情况、代码风格问题。

我只需要看它标出来的地方，不用逐行扫了。每周省大概 3 小时。

## 2. 多平台发布

一个 Markdown 文件，AI 改写成 Twitter thread、掘金文章、知乎文章、小红书帖子、英文博客。

一个输入，5 个以上输出。不用手动复制粘贴改格式。

## 3. 服务监控

一个"巡查 agent"：每 30 分钟检查服务状态，监控队列健康度，只在出故障的时候告警。

不用盯仪表盘。沉默就是正常。

## 4. 每日技术简报

每天早上，一个 agent 扫 GitHub Trending、HN 热门、AI 和开发相关的 RSS。

发一个 3 条摘要到 Telegram。5 分钟就能挑出写作选题，不用花 1 小时刷信息流。

## 5. 知识库自动整理

我学到的东西全扔进一个 Markdown 仓库。AI 自动打标签、分类、关联相关笔记。

6 个月下来：一个可搜索的个人 wiki。零手动归档。

## 6. 会议纪要提取

录音 → Whisper 转文字 → AI 提取决策、待办事项、跟进日期。自动推到任务管理工具。

## 7. 多 Agent 协调

上面这些不是各自独立的脚本。三个 AI agent 在协调运行：

- 🎯 小Z（规划）：拆解任务，分配工作
- 🔍 巡查（监控）：安静值守，有事才说
- 🔨 营造（执行）：写代码、搭项目、出结果

它们之间有通信。我只管定目标。

## 经验

别一口气全自动化。

挑一件每天最浪费时间的事。做出来，哪怕丑，能跑就行。然后再做下一个。

先从痛点最大的开始。
